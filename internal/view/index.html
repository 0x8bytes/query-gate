How to query

所有 /api/ 请求均需携带 X-API-Key: <key>。所有接口均为 POST，query 字段传 SQL 字符串（禁止 SELECT *，须列出列名）。Flow: 列库 → 查表结构 → 执行查询。支持驱动：mysql / postgres / mongodb。

# 1) 列出可查数据库
curl -XPOST -H "X-API-Key: $KEY" {{.BaseURL}}/api/v1/databases/list -d '{}'

# 2) 列出 "{{.ExampleDB}}" 的表／集合
curl -XPOST -H "X-API-Key: $KEY" {{.BaseURL}}/api/v1/tables/list -d '{"db":"{{.ExampleDB}}"}'

# 3) 查看表结构（DDL 或采样字段）
curl -XPOST -H "X-API-Key: $KEY" {{.BaseURL}}/api/v1/tables/detail -d '{"db":"{{.ExampleDB}}","names":["<table>"]}'

# 4) 执行只读查询（需列出列名，禁止 SELECT *）
curl -XPOST -H "X-API-Key: $KEY" {{.BaseURL}}/api/v1/query -d '{"db":"{{.ExampleDB}}","query":"SELECT id FROM t","limit":100}'

# MongoDB 查询：query 传 JSON 字符串（{"collection","pipeline"|"find"}）
# curl -XPOST -H "X-API-Key: $KEY" {{.BaseURL}}/api/v1/query \
#   -d '{"db":"{{.ExampleDB}}","query":"{\"collection\":\"users\",\"find\":{\"active\":true}}"}'

Exec (write, super_admin only)

/api/v1/exec 执行写操作（INSERT/UPDATE/DELETE/DDL），仅超级管理员的 key 可调，走该库的写账号（exec_dsn）。不经 guard，db 须已配置 exec_dsn。

# 写操作（仅 super_admin；db 须已配 exec_dsn）
curl -XPOST -H "X-API-Key: $KEY" {{.BaseURL}}/api/v1/exec \
  -d '{"db":"{{.ExampleDB}}","sql":"UPDATE t SET a=1 WHERE id=5"}'

Constraints

仅允许只读操作。guard 禁止：SELECT *、写操作/DDL/权限变更、多语句、INTO OUTFILE、EXPLAIN ANALYZE、FOR UPDATE、SLEEP/BENCHMARK/LOAD_FILE、敏感列（由管理员通过 admin API 配置黑名单）。唯一安全边界：DSN 使用只读数据库账号。
